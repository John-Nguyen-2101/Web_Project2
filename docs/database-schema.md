# Database Schema

Final Step 3 Supabase schema for the song import path.

## `songs`

| Column | Type | Notes |
| --- | --- | --- |
| `id` | `uuid` | Primary key, default generated by database. |
| `legacy_song_id` | `text` | Stable legacy import key. Required and unique. |
| `slug` | `text` | URL-safe song slug. Required and unique. |
| `title` | `text` | Song title. Required. |
| `author_name` | `text` | Artist, composer, or author display name. |
| `style` | `text` | Musical style or genre. |
| `recommended_tempo_text` | `text` | Human-readable tempo range. |
| `bpm` | `integer` | Default practice tempo. |
| `time_sig_top` | `integer` | Time signature numerator. |
| `time_sig_bottom` | `integer` | Time signature denominator. |
| `meter_mode` | `text` | Meter interpretation, for example `simple`. |
| `song_key` | `text` | Default key. |
| `scale` | `text` | Scale or mode. |
| `music_region` | `text` | Optional region/category metadata. |
| `uploaded_by` | `uuid` | User/profile id that owns the imported song. Required if the DB enforces ownership. |
| `status` | `text` | Editorial state, for example `draft` or `published`. |
| `visibility` | `text` | Access state, for example `private`, `unlisted`, or `public`. |
| `current_version_id` | `uuid` | Nullable foreign key to `song_versions.id`. Set after version import. |
| `view_count` | `integer` | Default `0`. |
| `like_count` | `integer` | Default `0`. |
| `comment_count` | `integer` | Default `0`. |
| `rating_avg` | `numeric` | Default `0`. |
| `rating_count` | `integer` | Default `0`. |
| `created_at` | `timestamptz` | Default `now()`. |
| `updated_at` | `timestamptz` | Default `now()`. |
| `published_at` | `timestamptz` | Nullable. |

Required import constraints:

- unique index/constraint on `songs.legacy_song_id`
- unique index/constraint on `songs.slug`
- nullable foreign key from `songs.current_version_id` to `song_versions.id`

## `song_versions`

| Column | Type | Notes |
| --- | --- | --- |
| `id` | `uuid` | Primary key, default generated by database. |
| `song_id` | `uuid` | Required foreign key to `songs.id`. |
| `version_number` | `integer` | Stable version number per song. Required. |
| `content_json` | `jsonb` | Parsed structured song content. Required. |
| `quick_text_legacy` | `text` | Original legacy quickText source. |
| `change_summary` | `text` | Import/change note. |
| `created_by` | `uuid` | User/profile id that created the version. Required if the DB enforces ownership. |
| `is_current` | `boolean` | Whether this is the current version. |
| `created_at` | `timestamptz` | Default `now()`. |

Required import constraints:

- unique index/constraint on `song_versions(song_id, version_number)`
- foreign key from `song_versions.song_id` to `songs.id`

## Import Ownership

The current migration output does not contain user ids. Before running `npm run import:songs`, provide one local server-only import owner id:

```bash
SUPABASE_IMPORT_USER_ID=00000000-0000-0000-0000-000000000000
```

The importer uses this value for both `songs.uploaded_by` and `song_versions.created_by`.
